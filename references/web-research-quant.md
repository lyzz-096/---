# 联网取证与量化规则

## 触发条件

当家长已经给出省份、科类/选科、分数或位次、专业/城市/就业偏好后，正式荐校前必须先查本地真实录取库，再联网查证。

如果学峰真实录取库存在，优先使用数据库结果；本地样例数据只能用于演示交互。

## 数据库优先流程

1. 用省份、分数/位次、专业关键词查学峰真实录取库。
2. 过滤“专业基本介绍”等非录取线数据，只保留投档线/专业分数线/录取线类记录。
3. 先展示数据库结果：学校、专业、年份、最低分、最低位次、来源文件。
4. 数据库没有命中时，明确说“数据库暂无记录”，再联网补查。
5. 数据库命中后仍可联网复核招生计划、章程和最新政策。

## 来源优先级

1. 省考试院/省教育考试院：一分一段表、投档线、征集志愿、批次线。
2. 高校本科招生网：分省分专业录取分数、招生计划、招生章程、选科要求。
3. 教育部/阳光高考：院校信息、专业目录、招生章程入口。
4. 可信第三方聚合页：只作为补充，必须标注“第三方口径，需复核”。

## 推荐前至少查什么

- 目标省份当年或最近一年一分一段表，用于分数转位次。
- 每个候选院校近 2-3 年同省同科类投档/专业录取最低位次。
- 每个候选专业当年招生计划变化：扩招、缩招、新增、停招。
- 院校层级与地域：985/211/双一流/省重点、城市就业环境。
- 专业画像：就业稳定性、薪资弹性、考公考编适配、读研必要性、行业周期风险。

## 搜索词模板

- `{省份} {年份} 一分一段表 {科类}`
- `{学校} 本科招生网 {省份} {专业} 录取分数 位次`
- `{学校} {年份} {省份} 招生计划 {专业}`
- `{学校} 招生章程 选科要求`
- `{专业} 就业方向 考公 考研 行业风险`

## 量化口径

### 候选生成顺序

先用分数/位次生成“足够宽”的候选池，再按家庭目标排序。不要只把往年分数线最接近的学校当作最优答案。

推荐时至少分清：

- 录取安全度：冲、稳、保。
- 目标适配度：就业、考公、深造、学校层级、城市、专业兴趣。
- 取舍代价：为了某个目标牺牲了什么。

### 位次安全度

对每个候选专业计算：

`位次差 = 往年最低位次 - 考生位次`

- 冲：位次差约 -8000 到 -1000
- 稳：位次差约 -1000 到 +3000
- 保：位次差约 +3000 到 +12000

如果近三年波动很大，档位下调半档或一档。

### 招生计划变化

- 扩招：风险略降
- 缩招：风险上升
- 新增专业：不能简单按老专业位次判断，必须单独提醒
- 停招/大类调整：不能直接比较，必须说明口径变化

### 专业画像

按家长诉求给权重：

- 稳定/国企：电气、自动化、能源、交通、师范、医学等方向加权；高波动互联网方向降风险说明。
- 高薪：计算机、电子信息、集成电路、软件、金融科技等方向加权，同时提示竞争和行业周期。
- 考公考编：法学、汉语言、财会、计算机、马克思主义、师范等方向加权。
- 读研深造：基础学科、医学、电子信息、计算机、自动化、材料化工等按读研必要性说明。
- 名校平台：985、211、双一流、省重点、推免资格和学科平台加权，同时提示地区和专业牺牲。
- 城市发展：目标城市产业、实习机会、回本省就业便利性加权。

## 输出格式

正式推荐必须包含：

1. 考生定位：省份、科类、分数、位次来源、偏好。
2. 数据来源：列出查到的网页来源、年份、口径。
3. 量化表：学校、专业、近年最低位次、位次差、计划变化、冲稳保档。
4. 路径建议：家长路线、学生路线、折中路线。
5. 风险提醒：数据年份、大小年、专业组/大类招生、计划变化、章程限制。

如果用户要求生成给家长看的方案，输出 HTML 或说明已生成 HTML 文件。HTML 中必须保留数据来源、待复核事项和冲稳保风险，不要只做漂亮展示。

## 禁止事项

- 不要说“数据库显示”除非确实有本地或官方数据库文件。
- 不要用没有来源的数字做正式推荐。
- 不要把第三方聚合数据当官方结论。
- 不要只给学校名，不解释专业、城市、就业/升学和风险。
